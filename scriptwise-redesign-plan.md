## 2. Hero 代码示例替换【P0】

### 【现状】

Hero 右侧代码窗口里展示的代码：

```
thresh(2,
  pk(Alice),
  pk(Bob),
  older(4320)
)
```

下方"花费路径分析"列出三条路径：`Alice + Bob 签名` / `Alice + 等待 30 天` / `Bob + 等待 30 天`。

### 【现状的问题】

1. `thresh` 比 `or` / `and` 抽象，第一眼看不懂是 N-of-M。
2. `older(4320)` 旁边写"30 天"——读者要知道 4320 ÷ 144 ≈ 30 才能理解关联。
3. 三条花费路径里有"Bob + 等待 30 天"——很多读者会怀疑是 bug。

### 【目标】

换一个**直观的、有真实生活语义的例子**：日常 Alice + Bob 共同签；万一 Bob 失联，Alice 等 30 天后可单独花。

### 【执行步骤】

1. 替换代码窗口里的 Miniscript 代码。
2. 替换"花费路径分析"下方的三条记录为两条记录。
3. 代码窗口顶部 `policy.miniscript` 标签和右下角 `P2WSH` 角标维持原样。

### 【最终代码（窗口内显示）】

```
or(
  and(pk(Alice), pk(Bob)),
  and(pk(Alice), older(4320))   // ≈ 30 天
)
```

颜色高亮规则：

- `or` / `and` → 橙色（`text-btc-500`）
- `pk(Alice)` / `pk(Bob)` → 黄色（`text-yellow-400`）
- `older(4320)` → 绿色（`text-emerald-400`）
- 注释 `// ≈ 30 天` → 灰色（`text-text-muted`）

### 【最终"花费路径分析"内容】

只保留两条：

| 圆点颜色 | 文案 |
| --- | --- |
| 绿色（`bg-emerald-400`） | `Alice + Bob 共同签名` |
| 橙色（`bg-btc-500`） | `Alice + 等待 30 天` |

### 【验收标准】

- [ ] 代码窗口正文是上面的 `or(and(...), and(...))` 三行 + 闭括号。
- [ ] 注释 `// ≈ 30 天` 出现在 `older(4320)` 同一行末尾。
- [ ] "花费路径分析"区域只有 2 条，文案完全一致。

     A：`Miniscript 的类型系统在编译期就能排除一类"链上无法被花费"的错误，且每个节点的可花费性、见证大小都可形式化分析。但任何复杂花费条件在投入真实资金前，都建议在 testnet/signet 上充分演练。`

  4. **Q：我必须懂底层 Bitcoin Script 才能用 Miniscript 吗？**
     A：`不需要。日常使用 Policy 这一层就够了——你描述"谁能花、什么时候花"，编译器负责生成对应的 Bitcoin Script。底层细节是工具与钱包的事。`

  5. **Q：Miniscript 与 Output Descriptor 是什么关系？**
     A：`Miniscript 描述的是"花费条件的结构"；Descriptor 在此基础上加上具体的密钥与派生路径，是钱包用来生成地址、构造交易的完整描述。可以理解为：Descriptor = Miniscript + 钥匙信息。`

  6. **Q：ScriptWise 是 Miniscript 的官方网站吗？**
     A：`不是。ScriptWise 是一个由社区个人维护的学习站点，目标是用真实场景帮助更多人理解 Miniscript。Miniscript 的规范与参考实现请以官方仓库为准。`

## 9. 动效与视觉节奏改版【P2】

### 【整体原则】

- 全站动效**不要超过两类**：滚动淡入 + 一处主打动画。
- 任何动画都不应该阻塞阅读：进入立刻可读，动画只是修饰。
- 移动端禁用复杂动画，保留淡入即可。

### 9.1 全局：scroll-triggered 淡入【必做】

每个 section 在进入视口时，做一次轻量的 fade + 上滑：

- 初始状态：`opacity: 0; transform: translateY(16px);`
- 进入视口后：`opacity: 1; transform: translateY(0);`，`transition: opacity 600ms ease-out, transform 600ms ease-out;`。
- 触发逻辑：用 `IntersectionObserver`，阈值 `0.15`，触发一次后不再恢复。
- **尊重用户偏好**：当 `prefers-reduced-motion: reduce` 时，**完全跳过**动画，元素直接以最终状态展示。

### 9.2 Hero 主打动画：Policy → Bitcoin Script 实时编译演示【强烈建议】

**为什么要做**：这是 Miniscript 这门技术最有说服力的一刻——用户亲眼看到一段 Policy 编译成一长串操作码。比任何文字都打动人。

> 重要前提：动画**不进行真实编译**。所有内容都是预录的静态文本，按计划好的时序逐字 / 逐行展示。下面给出的所有时间、文本、样式，执行端必须**逐字照做**，不得自行替换文案、调整时长或重排顺序。

#### 9.2.1 窗口结构

把 Hero 右侧"代码窗口"原本只有一段代码的区域，改造成"上下两个面板"。**保留**窗口顶部的红黄绿三圆点 + `policy.miniscript` 文件名 + 右下角 `P2WSH` 角标——它们都不变。

新结构（自顶向下）：

```
┌───────────────────────────────────────────────┐
│  ● ● ●   policy.miniscript                    │  ← 保留原 header，不变
├───────────────────────────────────────────────┤
│  PANEL A：Policy 面板                         │
│  （此处用打字机效果展示 Policy 代码）         │
├───────────────────────────────────────────────┤
│  ↓ compiles to                                │  ← 分隔条（见下文）
├───────────────────────────────────────────────┤
│  PANEL B：Bitcoin Script 面板                 │
│  （此处用逐行浮现展示操作码）                 │
├───────────────────────────────────────────────┤
│  示例：编译过程示意           [↻ 重播]        │  ← 底部状态栏
└───────────────────────────────────────────────┘
                                   ┌──────────┐
                                   │  P2WSH   │  ← 保留原右下角角标
                                   └──────────┘
```

样式细节：

- 上下两个面板之间用一条横向分隔条，文案 `↓ compiles to`，颜色 `text-text-muted`，字号 `text-xs`，字体 `font-mono`，居中显示，上下各 `py-3`。
- Panel A 高度大约容纳 4 行代码（即 Policy 代码加闭括号一共 4 行），加上 `p-5` 内边距。
- Panel B 高度大约容纳 6 行代码，加上 `p-5` 内边距。
- 两个面板的背景色相同（沿用原代码窗口背景 `bg-surface-card`），分隔条不要做单独的背景色。
- 底部状态栏左侧固定显示 `示例：编译过程示意`（颜色 `text-text-muted`，字号 `text-xs`），右侧放重播按钮（默认隐藏，详见 9.2.5）。

#### 9.2.2 Panel A 内容（Policy 文本）

**完整文本（4 行）**——执行端按这个原文一字不差地逐字打出：

```
or(
  and(pk(Alice), pk(Bob)),
  and(pk(Alice), older(4320))   // ≈ 30 天
)
```

颜色高亮（在动画完成后保持的最终状态）：

- `or` / `and` → 橙色 `text-btc-500`
- `pk(Alice)` / `pk(Bob)` → 黄色 `text-yellow-400`
- `older(4320)` → 绿色 `text-emerald-400`
- `// ≈ 30 天` → 灰色 `text-text-muted`
- 其它符号（括号、逗号、空格） → 默认色 `text-text-primary`

> 实现提示：可以把整段 Policy 拆成"带颜色 token 的数组"，逐字推进时按 token 染色；最简实现是先逐字以默认色写入，全部写完后再做一次性染色——两种都行，最终态保持上面的高亮规则即可。

#### 9.2.3 Panel B 内容（Bitcoin Script 文本）

**完整文本（6 行）**——执行端按这个原文逐行浮现：

```
OP_DUP <Alice> OP_CHECKSIGVERIFY
OP_IF
  <Bob> OP_CHECKSIG
OP_ELSE
  4320 OP_CSV
OP_ENDIF
```

颜色高亮（在动画完成后保持的最终状态）：

- `OP_*` 操作码 → 橙色 `text-btc-500`
- `<Alice>` / `<Bob>` → 黄色 `text-yellow-400`
- `4320` → 绿色 `text-emerald-400`
- 其它（缩进空格） → 默认色

> 这段 Bitcoin Script 仅作为"看起来对"的演示用，不要求严格等价于真实编译结果。不要替换、不要"修正"、不要补全。

#### 9.2.4 动画时序（精确到毫秒）

总时长 3000ms，分两阶段。所有时间从"动画开始"那一刻起算。

**阶段 1：Panel A 打字机（0 → 1500ms）**

- 把上面 Policy 文本的所有字符（包括换行、空格）拼成一个字符串，长度记为 `N`。
- 单位推进间隔 = `1500 / N` 毫秒（约 15–20ms / 字符）。
- 每个推进周期，向 Panel A 追加 1 个字符。换行字符 `\n` 触发换行，不消耗额外时间。
- 在当前位置末尾渲染一个**光标**：宽 `2px`、高 `1em`、颜色 `bg-btc-500`、`animate-pulse`（500ms 周期闪烁）。
- 阶段 1 结束时光标隐藏。

**阶段 2：Panel B 逐行浮现（1500 → 3000ms）**

- Panel B 共 6 行。
- 第 `i` 行（i 从 0 起）出现时间 = `1500 + i * 250` ms。即 1500、1750、2000、2250、2500、2750ms 各浮现一行。
- 每行的浮现动画：`opacity 0 → 1`，`translateY(8px) → translateY(0)`，时长 `300ms`，缓动 `ease-out`。
- 阶段 2 结束（3000ms 时刻）整个 Panel B 全部可见。

**阶段 0：进入触发**

- 用 `IntersectionObserver` 监听 Hero 代码窗口元素，阈值 `0.3`。
- 首次进入视口时，触发动画。**只触发一次**，再次滚出/滚入不会重新播放。
- 在动画触发前，Panel A、Panel B 都应保持"空白"状态（保留面板高度，避免布局跳动）——空白状态用一个 `min-height` 占位即可。

#### 9.2.5 重播按钮

- 桌面端，鼠标悬浮整个代码窗口时，底部状态栏右侧显示重播按钮：圆形或圆角矩形均可，文案 `↻ 重播`，字号 `text-xs`，颜色 `text-text-muted`，悬浮时变为 `text-btc-500`。
- 鼠标移出窗口后按钮淡出（`opacity 1 → 0`，200ms）。
- 点击重播按钮时：
  1. 立刻清空 Panel A 和 Panel B 的内容（回到空白状态）。
  2. 立刻重新执行阶段 1 + 阶段 2 的完整时序。
  3. 重播过程中按钮保持隐藏，重播完成后再次按 hover 规则显示。

#### 9.2.6 移动端与可访问性

- 视口宽度 `< 768px` 时：**完全跳过动画**，Panel A 直接展示完整 Policy 终态、Panel B 直接展示完整 Bitcoin Script 终态。不显示重播按钮。
- 当 `prefers-reduced-motion: reduce` 时：与移动端同样的处理——**完全跳过动画**，直接展示两个面板的终态，不显示重播按钮。
- 这两个判断必须在客户端组件首次挂载时就完成，**不能**先播一段再切换；否则会有"动了一下又跳到终态"的闪烁。

#### 9.2.7 实现技术建议

执行端可以自由选择实现方式，下面是参考方案：

- 用一个 React 客户端组件（如 `<HeroCompileAnimation />`）封装整个窗口。
- 用 `useState` 管理：`policyText`（已渲染的 Policy 字符串）、`visibleScriptLines`（已浮现的 Bitcoin Script 行数）、`hasPlayed`（是否已播放过，用于一次性触发）、`isReplaying`（当前是否在重播）。
- 用 `useEffect` + `IntersectionObserver` 监听首次进入视口。
- 用 `setTimeout` / `setInterval` 推进打字与逐行浮现；卸载时清理定时器。
- 用 `window.matchMedia('(prefers-reduced-motion: reduce)')` 与 `window.matchMedia('(max-width: 767px)')` 在挂载时判断是否跳过动画。
- 不要引入新的动画依赖（如 framer-motion）；用纯 CSS transition + state 切换即可。
