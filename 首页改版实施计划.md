## 一、改版总览

### 1.1 核心方向

将首页从「技术解释优先」调整为「意义与场景优先，技术解释后置」。改版围绕三个信息层次：

1. **Bitcoin 的 UTXO 可以被编程**（建立认知）
2. **Miniscript 是更高级、更清晰的表达方式**（建立价值感）
3. **这些能力能映射到现实场景**（建立关联感）

### 1.2 首页模块顺序变更

| 序号 | 改版前（当前） | 改版后（目标） |
|------|---------------|---------------|
| 1 | `HomepageHero` | `HomepageHero`（文案微调） |
| 2 | `IntroChallengeSection`（Bitcoin Script vs Miniscript 对比） | **🆕 `TransitionSection`**（新增过渡模块：单签 vs 多签） |
| 3 | `IntroCoreConceptsSection`（Policy / Miniscript / Descriptor + Technical Stack） | `IntroChallengeSection`（保持在 Transition 之后） |
| 4 | `IntroApplicationsSection` | **🆕 `MiniscriptBenefitsSection`**（可迁移性 + 可组合性） |
| 5 | `IntroWhyMattersSection` | `IntroApplicationsSection`（前置到技术栈之前） |
| 6 | `HomepageWallets` | `IntroCoreConceptsSection` + Technical Stack（后置） |
| 7 | 底部 CTA + Footer | `HomepageWallets` |
| 8 | — | 底部 CTA + Footer |

> **注意**：`IntroWhyMattersSection`（Why Miniscript Matters）在改版后**移除**。其内容（促进创新、企业应用、生态发展、效率提升）与改版后信息架构重叠——价值感已由过渡模块 + Applications 前置承担。若未来需要，可作为 `/resources` 页的补充内容。

#### 2.2.2 设计规格

- **背景**：`bg-surface-card`（与 ChallengeSection 一致），`border-t border-border-subtle`
